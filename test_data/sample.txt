Python es un lenguaje de programación de alto nivel creado por Guido van Rossum y lanzado por primera vez en 1991. La filosofía de diseño de Python enfatiza la legibilidad del código con su notable uso de sangría significativa. Sus constructos de lenguaje y enfoque orientado a objetos ayudan a los programadores a escribir código claro y lógico para proyectos pequeños y grandes.

Python tiene una biblioteca estándar extensa, frecuentemente citada como una de sus mayores fortalezas. La Python Software Foundation gestiona y dirige el desarrollo de Python. Python se usa ampliamente en desarrollo web, ciencia de datos, inteligencia artificial y computación científica.

Django es un framework web de alto nivel para Python que fomenta el desarrollo rápido y el diseño limpio y pragmático. Fue desarrollado originalmente por Adrian Holovaty y Simon Willison en 2005. Django sigue el patrón arquitectónico model-template-view y es mantenido por la Django Software Foundation.

FastAPI es un framework web moderno y rápido para construir APIs con Python basado en anotaciones de tipo estándar. Fue desarrollado por Sebastian Ramirez y lanzado en 2018. FastAPI está construido sobre Starlette para la capa web y Pydantic para validación de datos. FastAPI es utilizado por empresas como Microsoft, Uber y Netflix.

LangChain es un framework para desarrollar aplicaciones impulsadas por modelos de lenguaje grandes. Fue creado por Harrison Chase en 2022. LangChain proporciona abstracciones para cadenas, agentes y sistemas de recuperación. LangChain se integra con múltiples proveedores de LLM incluyendo OpenAI, Anthropic y Cohere.

Qdrant es un motor de búsqueda vectorial de código abierto escrito en Rust. Es desarrollado por Qdrant, Inc. y fue lanzado en 2021. Qdrant soporta búsqueda por similitud con varias métricas de distancia incluyendo coseno, producto punto y distancia euclidiana. Qdrant se usa para búsqueda semántica, sistemas de recomendación y aplicaciones RAG.

NebulaGraph es una base de datos de grafos distribuida construida para datos de grafos a gran escala. Es desarrollada por Vesoft, Inc. NebulaGraph usa una arquitectura shared-nothing y soporta el lenguaje de consulta nGQL. NebulaGraph se usa en redes sociales, detección de fraude y aplicaciones de grafos de conocimiento.

El error QDRANT_CONNECTION_TIMEOUT ocurre cuando el cliente Qdrant no puede conectarse al servidor dentro del tiempo de espera configurado. Este error afecta la versión 1.17 de Qdrant y tiene severidad alta. Los síntomas incluyen timeouts en consultas y fallos de ingesta. La causa raíz es usualmente una configuración incorrecta del host o puerto, o que el servicio no está en línea.

El error NEBULA_SESSION_POOL_EXHAUSTED ocurre cuando el pool de sesiones de NebulaGraph se agota. Afecta todas las versiones y tiene severidad crítica. Los síntomas incluyen errores de conexión intermitentes y degradación del rendimiento. La causa raíz es un pool de conexiones demasiado pequeño para la carga concurrente.

El error TRIPLET_EXTRACTION_FAILED ocurre cuando el LLM no produce triplets válidos desde un chunk. Afecta la versión 2.1 del pipeline de ingesta y tiene severidad media. Los síntomas incluyen documentos con cero triplets extraídos y warnings en los logs. La causa raíz es contenido de entrada ambiguo o respuestas del LLM que no contienen arrays JSON.

La incidencia QDRANT_CONNECTION_TIMEOUT está resuelta por el fix VERIFY_QDRANT_HOST el cual verifica la conectividad del host antes de iniciar la ingesta. El fix fue implementado en el paso 1 del playbook de diagnóstico. La incidencia NEBULA_SESSION_POOL_EXHAUSTED está resuelta por el fix INCREASE_POOL_SIZE el cual aumenta el tamaño del pool de sesiones y agrega reconexión automática. El fix fue implementado en el paso 3 del playbook de infraestructura.

La incidencia QDRANT_CONNECTION_TIMEOUT es escalada al equipo de Infraestructura cuando persiste después del fix VERIFY_QDRANT_HOST. La incidencia NEBULA_SESSION_POOL_EXHAUSTED es escalada al equipo de Plataforma cuando el fix INCREASE_POOL_SIZE no resuelve el problema. La incidencia TRIPLET_EXTRACTION_FAILED es gobernada por la política de calidad de datos la cual requiere validación del formato de entrada antes de la extracción.

El playbook de diagnóstico documentado en el manual de operaciones contiene los pasos para resolver el error QDRANT_CONNECTION_TIMEOUT. El playbook de infraestructura documentado en el wiki interno contiene los pasos para resolver el error NEBULA_SESSION_POOL_EXHAUSTED.
